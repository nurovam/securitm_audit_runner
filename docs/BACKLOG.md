# План Доработок

Этот файл фиксирует единый backlog по итогам внутреннего review и review из `securitm_audit_runner.pdf`.
Цель: идти по очереди, не смешивать архитектурные рефакторы с функциональными исправлениями и иметь
понятный порядок коммитов.

## Принципы

- Сначала исправляем дефекты, которые влияют на корректность проверок и выгрузку результатов.
- Потом доводим упаковку, архитектуру и безопасность реализации.
- Только после этого расширяем покрытие, тесты, CI и процессные улучшения репозитория.
- Для спорных пунктов review применяем прагматичный подход: для solo-проекта используем `main` + `feature/*`,
  а не обязательный full GitFlow.
