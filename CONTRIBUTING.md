# Contributing

Этот репозиторий ведётся в solo-режиме, но правила работы оформлены так, чтобы проект оставался
предсказуемым и пригодным для роста.

## Ветвление

- Основная ветка: `main`.
- Новая работа выполняется в коротких ветках `feature/*`, `fix/*`, `docs/*`, `chore/*`.
- Полный GitFlow с `develop` здесь не обязателен.

Примеры:

- `feature/fix-met-passwd-perms`
- `fix/pdf-optional-import`
- `docs/update-readme`

## Коммиты

- Используем короткие atomic commits.
- Формат сообщений: conventional commits.

Примеры:

- `fix: correct passwd/group permission mask`
- `feat: add tests for builtin checks`
- `docs: document manual checks policy`

## Порядок Работы

1. Синхронизировать `main`.
2. Создать короткую рабочую ветку.
3. Сделать одну логическую группу изменений.
4. Прогнать локальные проверки.
5. Слить изменения обратно в `main`.

## Минимальный Локальный Чек Перед Commit

- Проект устанавливается в чистом виртуальном окружении.
- CLI запускается без падения.
- Если установлены dev-зависимости, проходит `python -m ruff check .`.
- Тесты проходят локально (`pytest`).
- Изменённые проверки имеют тесты или хотя бы smoke-проверку.
- README/конфиг обновлены, если менялось поведение пользователя.

## Артефакты И Данные

- Генерируемые отчёты (`audit-report.json`, `audit-report.pdf`) в git не коммитятся.
- Вспомогательные PDF и справочные материалы добавляются в репозиторий только осознанно.
- Секреты и токены хранятся только в переменных окружения.

## Релизы

- Релизы помечаются git-тегами вида `vX.Y.Z`.
- Перед постановкой тега нужно убедиться, что README, changelog и конфиг актуальны.
